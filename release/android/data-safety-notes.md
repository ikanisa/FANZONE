# Google Play Data Safety Notes

Use these notes to complete Play Console Data safety. Verify against the final production build and SDK list before submission.

## Data Collected

| Data type | Collected | Shared | Purpose |
| --- | --- | --- | --- |
| Phone number | Yes | No sale | WhatsApp OTP authentication and account recovery. |
| User IDs | Yes | No sale | Autogenerated 6-digit FANZONE ID, Supabase user ID, support, audit, wallet ledger. |
| App activity | Yes | No sale | Venue browsing, orders, pools, games, teams, wallet activity, eligibility, support. |
| App info and performance | Yes | No sale | Runtime errors, stability, diagnostics, release quality. |
| Device or other IDs | Yes | No sale | Firebase Cloud Messaging tokens and app session operation. |
| Financial info | No direct payment data | No | The app does not process external payment credentials or bank/card details. |
| Location | No | No | Do not declare precise or approximate location unless a future release requests OS location permission. |
| Contacts | No | No | The app does not request contact access. |
| Photos/videos/audio | No | No | The app does not request media/camera/microphone access in the current mobile release. |

## Security Practices

- Data is encrypted in transit with HTTPS/TLS.
- Account deletion is available through support and in-app request flows where enabled.
- Wallet, order, pool, game, and audit records may be retained for fraud prevention, ledger integrity, disputes, and legal compliance.

## Important Declaration Notes

- FET is a closed-loop reward point, not cash, not an investment asset, and not withdrawable.
- The app does not contain real-money wagering or gambling mechanics.
- External payment instructions may show MoMo USSD, Revolut links, cash, card, or other venue-supported instructions, but FANZONE does not process payments.
- If analytics, crash reporting, or advertising SDKs are added later, update this file and the Play Console form before submission.
